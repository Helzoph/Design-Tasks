# 项目代码结构

## 目录结构

```
<项目名称>
├── docs/
│   ├── design/  # 蓝图：系统应该是什么样子
│   │   ├── backend/  # 后端设计
│   │   │   ├── API-Reference/  # [关键] API 接口定义
│   │   │   ├── Architecture/  # 宏观架构图、技术选型、服务划分
│   │   │   ├── assets/  # 设计相关的资源，如架构图源文件
│   │   │   ├── Data-Model/  # [关键] 数据库模型、实体关系图 (ERD)、数据结构设计
│   │   │   │   ├── Entities/  # 实体模型定义
│   │   │   │   ├── Relationships/  # 实体间关系
│   │   │   │   └── Schemas/  # 数据模式定义
│   │   │   ├── Features/  # 按功能模块描述详细的业务逻辑和处理流程
│   │   │   ├── Guides/  # 开发指南，如错误码定义、认证流程等
│   │   │   └── Testing/  # 后端测试项，描述需要测试的场景、策略和范围
│   │   └── frontend/  # 前端设计
│   │       ├── Architecture/  # 宏观架构、状态管理方案、目录结构规范
│   │       ├── assets/  # UI设计稿、图标等
│   │       ├── Components/  # [关键] 组件设计文档
│   │       ├── Data-Structures/  # [关键] 前端数据结构设计
│   │       │   ├── State/  # 状态管理结构
│   │       │   ├── Props/  # 组件属性类型定义
│   │       │   └── Interfaces/  # 接口类型定义
│   │       ├── Guides/  # 开发指南，如代码风格、API调用封装策略
│   │       ├── Style-Guide/  # 视觉设计规范，如颜色、字体、间距
│   │       └── Testing/  # 前端测试项，描述需要测试的场景、策略和范围
│   └── tasks/  # 工单：如何将代码变成蓝图的样子
├── src/  # 源代码
│   ├── backend/  # 后端源代码
│   └── frontend/  # 前端源代码
├── tests/  # 测试文件
├── scripts/  # 脚本工具
└── README.md  # 项目说明
```

## 关键设计文档说明

### 数据结构设计（必须完整覆盖）

**后端数据结构设计必须包括：**
- 所有实体（Entity）的完整定义
- 数据库表结构（字段、类型、约束）
- 实体间关系（1:1、1:N、N:M）
- 索引设计
- 数据流转逻辑

**前端数据结构设计必须包括：**
- 状态管理结构设计
- 组件 Props/State 接口定义
- API 数据结构映射
- 缓存数据结构

**完整性要求：** 任何一个数据结构都不应该在文档中缺失。拿着这套文档，应该能够复刻整个数据模型。
